# タロット占い運用システム 仕様書

| 項目 | 内容 |
|---|---|
| システム名 | トワイライト・タロット（AI × 占い副業 運用システム） |
| 目的 | 霊視鑑定士として、集客〜鑑定〜顧客管理を最小の手間で回し、無料鑑定から有料鑑定・リピートへつなぐ |
| 利用者 | 運営者本人（1名・自分専用ツール） |
| 版 | 1.1 |
| 作成日 | 2026-07-20 |

---

## 1. 概要

本システムは、スマホ1台で「タロット占い副業」を運用するための仕組み一式です。動画で解説されているマーケティング設計（無料鑑定で全力回答し信頼を築く → 根本の“次なる課題”を提示 → 有料鑑定・リピートにつなぐ）をベースに、AIツールと管理ツールを役割分担させて構成しています。

設計の基本思想は **「AI生成はチャットに任せ、ダッシュボードは準備とデータ管理に専念する」ハイブリッド構成** です。これにより、生成の柔軟さ（チャット）と、記録の永続性（ダッシュボード）を両立します。

---

## 2. システム構成

### 2.1 全体像

本システムは次の4つの構成要素からなります。

| # | 構成要素 | 形態 | 主な役割 |
|---|---|---|---|
| A | トワイライト・タロット ダッシュボード | Webアプリ（単一HTMLファイル） | カードを引く／投稿・鑑定の指示づくり／顧客・売上管理／ブランド設定 |
| B | Claude 鑑定プロジェクト | Claudeのプロジェクト（指示文） | 鑑定文の生成 |
| C | ChatGPT 投稿＆画像GPT | ChatGPTのCustom GPT（指示文） | Threads投稿文・画像の生成 |
| D | 初回DM返信テンプレ集 | ドキュメント（Markdown） | 申し込み〜鑑定結果〜有料誘導のDM文面 |

### 2.2 連携フロー（業務フロー）

```
① 集客        ChatGPT(C) で投稿・画像を作成 → Threadsへ投稿
② 申し込み    コメント/DMで「鑑定」希望が届く
③ 受付        DMテンプレ(D)①でスピード返信 → 悩みをヒアリング
④ カードを引く ダッシュボード(A) でスプレッド選択・カードを引く → 入力テキストをコピー
⑤ 鑑定文生成  Claude(B) に④を貼り付け → 鑑定文を生成
⑥ 鑑定書作成  Canvaで「鑑定書」に整える
⑦ 結果送付    DMテンプレ(D)③で鑑定書＋添え文を送付（解決策＋次なる課題）
⑧ 有料誘導    希望が出たらDMテンプレ(D)④で有料メニューを案内
⑨ 記録        ダッシュボード(A) の顧客・売上管理に記録
⑩ リピート    DMテンプレ(D)⑤でフォロー → ⑤へ戻り深掘り鑑定
```

## 3. 構成要素A：ダッシュボード仕様

### 3.1 技術仕様

| 項目 | 内容 |
|---|---|
| 実装 | 単一HTMLファイル（HTML / CSS / JavaScript、フレームワーク不要・ビルド不要） |
| データ保存 | ブラウザの永続ストレージ（端末ローカル・本人のみ）。localStorageに保存 |
| 外部通信 | なし（AI生成は行わない） |
| 起動方法 | ファイルをダウンロードし、スマホ・PCのブラウザで開く（ホーム画面に追加すればアプリのように利用可能） |
| 対応環境 | スマホ・PCのブラウザ（レスポンシブ） |
| デザイン | 深いプラム × アンティークゴールド。見出し=明朝体（Shippori Mincho）、本文=Zen Kaku Gothic New |

### 3.2 画面構成（4タブ）

#### タブ1：カードを引く
- スプレッドを7種から選択。
- 「カードを引く」で、選んだスプレッドの枚数分を正位置／逆位置ランダムで引く。引き直し可能。
- 鑑定モード（無料／有料）、お名前（任意）、お悩みを入力。
- 上記から **「Claudeに貼る入力」テキスト** を自動生成し、コピーできる。

**対応スプレッド**

| スプレッド | 枚数 | ポジション |
|---|---|---|
| ワンオラクル | 1 | 今のあなたへのメッセージ |
| スリーカード（時間の流れ） | 3 | 過去／現在／未来 |
| スリーカード（問題解決） | 3 | 現状／課題／アドバイス |
| 相手の気持ち | 3 | 相手の本音／相手から見たあなた／二人の行方 |
| 二者択一 | 3 | あなたの現状／選択肢Aの結果／選択肢Bの結果 |
| ヘキサグラム | 7 | 過去／現在／未来／周囲／相手／助言／最終結果 |
| ケルト十字 | 10 | 現在／課題／過去の要因／近い過去／顕在意識／近い未来／本音／周囲／希望と不安／最終結果 |

**デッキ**：タロット78枚（大アルカナ22枚＋小アルカナ56枚：ワンド／カップ／ソード／ペンタクルの4スート × エース〜10・ペイジ・ナイト・クイーン・キング）。

#### タブ2：投稿メーカー
- 投稿タイプ（寄り添い系／鑑定誘導系）、ジャンル、季節・イベント、本数（1〜5）を選択。
- 上記から **「ChatGPTに貼る指示」テキスト** を自動生成し、コピーできる。
- 季節・イベントを「なし」にすると、ChatGPT側で反応が取れそうなイベントを検索・提案する運用。

| 選択項目 | 選択肢 |
|---|---|
| 投稿タイプ | 寄り添い系／鑑定誘導系 |
| ジャンル | 恋愛／仕事・お金／人間関係／オールジャンル |
| 季節・イベント | なし／新月／満月／七夕／季節の変わり目／年末年始／バレンタイン |
| 本数 | 1〜5本 |

#### タブ3：顧客・売上管理
- 顧客の追加・編集・削除。
- 集計を自動表示：総売上／有料客数／有料転換率／累計リピート回数。

**顧客レコードのデータ項目**

| 項目 | 型 | 説明 |
|---|---|---|
| 名前 | テキスト | ハンドルネーム等 |
| 流入日 | 日付 | 申し込み日 |
| ステータス | 選択 | 見込み客／無料鑑定済／有料客 |
| メニュー | テキスト | 提供した鑑定メニュー |
| 売上 | 数値（円） | その顧客の売上 |
| リピート回数 | 数値 | 有料鑑定の回数 |
| メモ | テキスト | 悩みの傾向・次回の切り口 |

#### タブ4：ブランド設定
- ブランドボイス（霊視鑑定士のキャラ設定・プロフィール）を編集・保存。
- デフォルトの悩みジャンルを設定。
- ここが「原本」となり、Claude(B)・ChatGPT(C)双方に貼り付ける元ネタになる。

### 3.3 データ保存仕様

| 保存キー | 内容 | スコープ |
|---|---|---|
| tarot:persona | ブランドボイス設定 | 端末ローカル・本人のみ |
| tarot:genre | デフォルトの悩みジャンル | 端末ローカル・本人のみ |
| tarot:customers | 顧客レコード一覧 | 端末ローカル・本人のみ |

> データは利用中の端末・ブラウザのlocalStorageに保存されます。別端末・別ブラウザとは共有されず、ブラウザのデータを消去すると失われるため、重要なデータは別途バックアップを推奨します。

## 4. 構成要素B：Claude 鑑定プロジェクト仕様

| 項目 | 内容 |
|---|---|
| 実装 | Claudeの「プロジェクト」に指示文を登録 |
| 入力 | ダッシュボード（タブ1）が出力する「Claudeに貼る入力」テキスト |
| 出力 | 鑑定文（本文のみ） |

**入力フォーマット**：モード／スプレッド／引いたカード（正逆）／お名前／お悩み。

**鑑定文の構成**：①寄り添いの導入 → ②カードごとの読み解き → ③総合鑑定 → ④根本にある課題の提示 → ⑤前向きな締めと有料への自然な誘い。

**モード別の分量**

| モード | 目安 | 方針 |
|---|---|---|
| 無料 | 500〜800字 | 出し惜しみせず全力回答＋「次なる課題」で締める |
| 有料 | 1000字以上 | 深掘り・心理やパターンの解説・具体的行動まで踏み込む |

---

## 5. 構成要素C：ChatGPT 投稿＆画像GPT仕様

| 項目 | 内容 |
|---|---|
| 実装 | ChatGPTのCustom GPTに指示文を登録（Web検索・画像生成をオン） |
| 入力 | ダッシュボード（タブ2）が出力する「ChatGPTに貼る指示」テキスト |
| 出力 | Threads投稿文（フック→本文→ハッシュタグ）／投稿・鑑定書用の画像 |

**投稿タイプ**
- 寄り添い系：共感・励ましで信頼を貯める（売り込みなし）。
- 鑑定誘導系：無料鑑定→DM【鑑定】へ誘導（例：「本日5名限定 無料鑑定」）。

**画像**：ブランドの世界観（プラム×ゴールド・月・星・タロット）で背景を生成。**日本語の文字は画像に描かせず、Canvaで載せる**運用（鑑定書もCanvaで仕上げる）。

## 6. 構成要素D：初回DM返信テンプレ集仕様

申し込み〜有料誘導〜フォローの5テンプレで構成。

| # | テンプレ | 用途 |
|---|---|---|
| ① | 受付 | 申し込み直後の即レス・悩みヒアリング |
| ② | ヒアリング受け | 悩みを受け止め、鑑定に入る前振り |
| ③ | 鑑定結果送付 | 鑑定書＋添え文（解決策＋次なる課題）★最重要 |
| ④ | 有料鑑定案内 | メニュー・料金の提示 |
| ⑤ | フォロー | 後日の様子伺い・リピートの種まき |

**運用の鉄則**：スピード返信／名前を必ず入れる／出し惜しみしない。

## 7. マーケティング設計（本システムの核）

1. **信頼構築**：無料鑑定でも全力で答え、「本当に親身になってくれる」という信頼をつくる。
2. **解決策＋次なる課題**：鑑定の最後に、お客様がまだ気づいていない根っこのテーマを提示する。
3. **自然な動機付け**：「根本もこの人に見てほしい」という気持ちが生まれ、有料鑑定・リピートにつながる。

これにより、フォロワーが多くなくても1人あたりの関係を深め、リピート率を高めることを狙う。

---

## 8. 非機能・制約事項

- ダッシュボードは端末ローカル（localStorage）保存のため、端末・ブラウザを変えたり、ブラウザのデータを消去するとデータは引き継がれない（要バックアップ）。
- 鑑定・投稿の生成はチャット（B・C）側に依存する。各サービスの利用条件・料金は別途必要。
- Custom GPTの作成には有料プランが必要な場合がある。
- 画像生成は日本語テキストが崩れやすいため、文字はCanvaで載せる。

---

## 9. 安全・コンプライアンス方針

全構成要素に共通して、以下を必須ルールとする。

- **誇大表現をしない**：実績・経歴は事実の範囲で記載する（誇大広告はアカウント停止や信頼失墜のリスク）。
- **断定的な保証をしない**：「必ず復縁する」等の確約はしない。占いは指針・寄り添いとして提供する。
- **不安を過度に煽らない**：脅し文句で契約を迫らない。有料への誘導は「気づき」と「希望」で行う。
- **深刻な悩みへの配慮**：強い落ち込み・自傷の気持ち・生活破綻など深刻なサインが見えた場合は、占いは寄り添いにとどめ、専門家や信頼できる人・相談窓口を頼るようやさしく促す。
